Local storage is a web storage method that helps us store data on the client’s computer in the form of key/value pairs in a web browser. The data is stored in local storage for a lifetime unless the user manually deletes it from the browser. It does not expire even when the user closes the window or tab. Instead, the data remains in the browser until and unless the browser's memory is cleared.
Session storage is very similar to the local storage. Still, the main difference lies in the lifespan as it persists in the browser until its current tab is on. Once you close the tab or terminate it, the data on session storage also gets lost. We can also set and retrieve its data using setItem() and getItem() methods, respectively, similar to the local storage methods.

A variable’s scope is the part of a program where it is available for use. JavaScript variables are lexically scoped, meaning that we can determine a variable’s scope from where it is declared in the source code.We can create variables with the let, const, and var declarations.In ES6 let and const, block scoping works as block act.we can not use these two variables outside their scope. However, var behaves differently.var can be access anywhere in the javascript file.

The event loop is the secret behind JavaScript’s asynchronous programming. JS executes all operations on a single thread, but using a few smart data structures, it gives us the illusion of multi-threading. An event loop is something that pulls stuff out of the queue and places it onto the function execution stack whenever the function stack becomes empty.

Undefined is a primitive value in JavaScript. A variable or an object has an undefined value when no value is assigned before using it. We can get undefined value when we call a non-existent property or method of an object or variables.
